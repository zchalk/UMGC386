Implementation

SASS implementation takes place in either the client or server side.

  • Client Side

    • Client side implementation typically takes works through Browser Execution. Sass is compiles in the browser using JS libraries. This allows for dynamic compilation, or changes to compile in the browser in real time.

  • Server Side

    • On the server side, SASS files are precompiled into CSS before deployment. This approach allows for better optimization and stability, by dynamic changes are not possible.

SASS

SASS operates by extending CSS through the addition of certain features and mechanisms.

  • Variables

    • Variables in sass allow you to save reused values under a term for ease of use. For example a background color might be saved as 'main background'

  • Nesting

    • SASS nesting is very similar to HTML nesting allowing for easier DOM traversal in the style sheet.

  • Mixins

    • Mixins are like variables except they hold entire groups of CSS declarations for reuse.

  • Functions

    • Functions allow for more dynamic calculations to occur without the use of JS. This leads to lighter weight webpages with higher responsivness.
